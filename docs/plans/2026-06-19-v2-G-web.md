# Sub-plan G: Web Frontend Updates

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Update the web frontend (Jinja2 + HTMX) to use the new API endpoints — `GET /jobs` for the dashboard, `GET /jobs/{id}` for job detail, and the segment audio player in the SRT viewer that calls `GET /jobs/{id}/segment/{index}/audio` and plays the presigned URL.

**Architecture:** Minimal changes — add new HTMX partial routes in `web/main.py` that call the updated API, update `dashboard.html` to render the new job data shape, and extend `srt_viewer.html` with a click-to-play audio player.

**Tech Stack:** Jinja2, HTMX, httpx (web → api calls), vanilla JS (audio player)

**Depends on:** Sub-plan E (POST /jobs), Sub-plan F (correction + clip endpoints), Sub-plan C (GET /jobs/{id})

## Global Constraints

- Web layer calls API via `API_URL` env var (`http://api:8080` in Docker, `http://localhost:8080` in dev)
- No additional JS libraries — use HTML5 `<audio>` element with presigned URL as `src`
- Presigned URL for clip expires in 600s — refresh on `<audio>` error event
- The upload form (already in web) calls `POST /jobs` after multipart-to-MinIO; ensure the new endpoint shape is used
- Correction UI (FR4) is a stretch goal for this sub-plan; core deliverable is dashboard + audio playback

@app.get("/partial/jobs", response_class=HTMLResponse)
async def partial_jobs(request: Request):
    async with httpx.AsyncClient() as client:
        resp = await client.get(f"{API_URL}/jobs")
    overview = resp.json()
    return templates.TemplateResponse("partials/jobs.html",
                                      {"request": request, **overview})
```

@app.get("/jobs/{job_id}/segment/{index}/play-url")
async def segment_play_url(job_id: str, index: int):
    """Proxy the presigned clip URL from the API (avoids CORS issues in the browser)."""
    async with httpx.AsyncClient() as client:
        resp = await client.get(f"{API_URL}/jobs/{job_id}/segment/{index}/audio")
    return resp.json()
```

- [ ] **Step 4: Update srt_viewer.html — add per-segment play button**

In the segment rendering loop in `srt_viewer.html`, add a play button and a shared `<audio>` element:

```html
<!-- At the top of the viewer, one shared audio element -->
<audio id="segment-player" controls style="position:sticky;top:0;width:100%"></audio>

<!-- In the segment loop, add play button -->
{% for seg in segments %}
<div class="segment" id="seg-{{ loop.index0 }}">
  <div class="segment-meta">
    <span class="ts">{{ seg.start | format_ts }} → {{ seg.end | format_ts }}</span>
    <button
      class="play-btn"
      data-job-id="{{ job_id }}"
      data-segment-index="{{ loop.index0 }}"
      onclick="playSegment(this)"
      title="Play segment">▶</button>
  </div>
  <p class="segment-text">{{ seg.text }}</p>
</div>
{% endfor %}

<script>
async function playSegment(btn) {
  const jobId = btn.dataset.jobId;
  const idx   = btn.dataset.segmentIndex;
  const resp  = await fetch(`/jobs/${jobId}/segment/${idx}/play-url`);
  const data  = await resp.json();
  const audio = document.getElementById('segment-player');
  audio.src = data.url;
  audio.play();
  // Re-fetch URL on error (presigned URL may have expired)
  audio.onerror = () => playSegment(btn);
}
</script>
```
